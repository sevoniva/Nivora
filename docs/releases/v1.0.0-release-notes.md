# Future v1.0.0 Release Notes Draft

These release notes are a future draft for maintainer review. They do not mean `v1.0.0` has been achieved. Nivora is currently a hardened beta-candidate foundation and is not production-ready.

## Summary

The future `v1.0.0` release target is to establish a reviewed open-source delivery control-plane foundation. The current repository focuses on clear architecture boundaries, local verification, documented operations, secure defaults, auditable runtime behavior, and guarded delivery workflows, but several major runtime and integration areas remain beta, experimental, foundation-only, or placeholder.

## Candidate Scope For Future GA Review

- Architecture guardrails and modular monolith boundaries.
- Documentation, examples, tutorials, and release playbook.
- CLI-driven local verification paths.
- OpenAPI/AsyncAPI parsing and API inventory discipline.
- Secure-default review for production-shaped config.
- Secret scan and redaction test coverage.

## Beta Scope

- Shell PipelineRun runtime and runner protocol foundations.
- PostgreSQL persistence foundation for critical PipelineRun runtime entities.
- Kubernetes YAML planning, dry-run, guarded apply, inventory, health, diff, and rollback foundations.
- Release orchestration and artifact binding foundations.
- OCI digest resolution foundation.
- Observability, diagnostics, metrics, runbooks, and packaging foundations.

## Experimental Scope

- GitOps / Argo CD sync foundations.
- DevSecOps external integrations.
- Cloud provider inventory adapters.
- Host deployment remote execution.
- Minimal web console.
- Full external plugin execution.

## Security Notes

- `configs/production.example.yaml` keeps auth enabled.
- Secret values are represented by SecretRef/CredentialRef metadata and must not be returned by normal APIs.
- Runner tokens and API tokens are stored as hashes.
- Kubernetes apply, Argo CD sync, Git push, host remote deploy, prune, and rollback execution remain explicit guarded operations.
- Threat model and security checklist are available under `docs/security/`.

## Known Limitations

- Nivora is currently not production-ready.
- Nivora is not a full distributed workflow engine.
- Nivora does not ship a Kubernetes operator.
- GitOps/Argo CD support is guarded and not production automation by default.
- Host deployment and cloud inventory remain experimental.
- Full enterprise SSO, Vault/KMS lifecycle hardening, SBOM/signing, external scanners, and ITSM integrations remain future work.
- Runner sandboxing requires operator-controlled isolation for untrusted workloads.

## Verification Target

Required:

```sh
make verify
git diff --check
```

Recommended where tooling and network access are available:

```sh
make docker-build
docker compose -f deployments/docker-compose/docker-compose.yaml config
make helm-template
make helm-lint
```
