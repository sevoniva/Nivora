# v1.0.0 GA Capability Matrix

This matrix is the final Phase 10.0 GA readiness view. It is intentionally honest: `v1.0.0` can be tagged only if maintainers accept the production-ready scope and the experimental limitations below.

## Status Legend

- **Production-ready**: stable enough for the documented use case when installed and operated according to the GA docs.
- **Beta**: usable and tested, but not the final production contract for every environment.
- **Experimental**: available for evaluation, local validation, or guarded operation only.
- **Planned**: roadmap item with model or documentation only.
- **Unsupported**: intentionally not provided by this release.

## Capability Matrix

| Area | GA Status | Production Scope | Limitations |
| --- | --- | --- | --- |
| Architecture guardrails | Production-ready | Modular monolith boundaries, domain/usecase/adapter separation, architecture verification | Large design changes still require RFC/ADR review |
| CLI and local verification | Production-ready | `nivora` CLI basics, local pipeline/deployment/release/security examples, config validation | UX is developer/operator focused, not a polished end-user product |
| Documentation and examples | Production-ready | README, user/operator/developer guides, tutorials, operations docs, release playbook, example validation | Docs must stay synchronized with future API changes |
| OpenAPI / AsyncAPI hygiene | Production-ready | Specs parse and are part of verification | Some placeholder route groups remain documented as not implemented |
| PipelineRun shell runtime | Beta | Shell PipelineRun execution, logs, events, audit, timeline, retry, timeout, cancellation, smoke tests | Not a distributed workflow engine; no privileged execution guarantee |
| Durable runtime recovery | Beta | PipelineRun/runner/outbox recovery foundations and reconciliation docs | DeploymentRun and ReleaseExecution recovery remain partial |
| Runner fleet protocol | Beta | runner identity, hashed tokens, heartbeat, claim, leases, logs, status, cancellation observation | No autoscaling or strong untrusted workload sandbox guarantee |
| PostgreSQL persistence | Beta | PipelineRun, JobRun, LogChunk, Event, AuditLog, Runner, outbox, idempotency foundation | DeploymentRun, Release, Artifact, Credential, and PolicyResult persistence coverage remains partial |
| Kubernetes YAML deployment | Beta | static YAML render, validation, server-side dry-run/apply foundations, inventory, health, diff, guarded rollback | Helm/Kustomize/CRD health and destructive prune are not GA production features |
| GitOps / Argo CD | Experimental | plan/diff/local working tree, status model, guarded sync request | No production sync automation, app creation, repo/cluster registration, or full Git provider integration |
| Artifact / OCI registry | Beta | OCI parsing, digest resolution foundation, digest-bound ReleaseArtifact, mutable-tag warnings, explicit insecure registry config | Full Harbor/Nexus/JFrog management APIs, signing, and scanning are not GA features |
| Release orchestration | Beta | ReleasePlan/ReleaseExecution, sequential multi-target local orchestration, aggregate status | Release trains, promotion workflows, and full approval workflows remain future work |
| DevSecOps policy gates | Experimental | SecurityScan/Finding model, fake/noop scanners, built-in rules, audit/events | Required Trivy/Cosign/SBOM/OPA/Kyverno integrations are not implemented |
| Secrets and credentials | Beta | SecretRef/Credential metadata, redaction, builtin provider, external provider skeletons | Production Vault/KMS lifecycle and secret rotation enforcement remain future hardening |
| Auth / RBAC | Beta | dev/token auth, OIDC provider foundation, service accounts, hashed API tokens, selected permission checks | Browser login UI, final enterprise SSO lifecycle, and complete multi-tenant enforcement remain future work |
| Audit and evidence | Beta | structured audit foundations, audit search, evidence bundle/export model, retention metadata | Tamper-evident external audit storage and retention jobs are not GA features |
| Observability and operations | Beta | request/correlation IDs, diagnostics, metrics, runbooks, SLO guidance, backup/restore docs | Bundled dashboards and distributed tracing export remain future work |
| Packaging | Beta | Dockerfile, Docker Compose local stack, Helm chart, Kubernetes manifests, config examples | No Kubernetes operator or managed production installer |
| Web console | Experimental | minimal web UI foundation consuming backend APIs | Not a complete production UI |
| Cloud inventory | Experimental | fake providers, AWS/Aliyun/Tencent skeletons, inventory models | No cloud deployment or complete provider coverage |
| Host deployment | Experimental | model, plan, dry-run/noop, guarded SSH surface | Remote SSH deployment is disabled by default and not GA production hardened |
| Plugin system | Beta | manifest, capability registry, API version, compatibility checks, templates | No marketplace or unsafe dynamic loading |

## Unsupported In v1.0.0

- Temporal, Tekton, Argo Workflows, or external workflow-engine integration.
- Default destructive Kubernetes prune/delete.
- Production Argo CD automation by default.
- Full GitHub/GitLab/Gitea provider integration.
- Full Harbor/Nexus/JFrog management integration.
- Required external scanners/signers/SBOM generators.
- Cloud deployment automation.
- Complete enterprise SSO and browser login.

## GA Interpretation

`v1.0.0` is a GA release for the documented control-plane foundation, local verification paths, architecture contracts, APIs/spec discipline, examples, operations documentation, and guarded backend runtime foundations. It is not a claim that every modeled integration is production-grade.
