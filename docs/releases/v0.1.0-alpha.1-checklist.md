# v0.1.0-alpha.1 Release Checklist

This checklist tracks the public alpha release foundation. It is intentionally honest about unsupported capabilities. Nivora is not production-ready.

## Completed Checks

- [x] README states alpha limitations and non-production status.
- [x] Roadmap covers phases through packaging and alpha hardening.
- [x] Capability matrix exists.
- [x] Alpha demo guide exists.
- [x] Changelog and release template exist.
- [x] Version target is `0.1.0-alpha.1`.
- [x] `make verify` runs local runtime, deployment, release, security, host, and web checks.
- [x] Secret scan runs through `./scripts/verify-no-secrets.sh`.
- [x] Architecture boundary check runs through `./scripts/verify-architecture.sh`.
- [x] Helm template/lint checks are available.
- [x] Dockerfile and Compose assets exist.

## Unsupported Or Partial Capabilities

- Production Kubernetes apply semantics.
- Destructive rollback execution.
- Production Argo CD automation.
- Real cloud deployment.
- Real Git provider write integration.
- Full Harbor/Nexus/JFrog management integration.
- External scanner, signer, SBOM generator, and ITSM integrations.
- Production SSO/OIDC.
- Vault/KMS-backed secret management.
- Complete product frontend.

## Manual Validation Notes

- `make docker-build` depends on Docker Hub and distroless registry reachability.
- Local Kubernetes, Argo CD, Harbor, Nexus, Gitea, and GitLab-lite environments are optional and are not required for the alpha demo or CI.
- Apply-oriented examples require explicit flags or environment variables and are not part of the default alpha path.

## Release Blockers

- Any committed secret or realistic credential.
- Any production-readiness claim.
- Any required external service in default tests.
- Failing `make verify`.
- Broken Helm template rendering when Helm is available.
