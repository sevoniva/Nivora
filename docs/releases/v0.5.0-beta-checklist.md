# v0.5.0-beta Readiness Checklist

This checklist is the Phase 9.0 beta-freeze baseline. It is not a GA release checklist and does not claim production readiness.

## Freeze Status

- [x] Feature expansion is frozen for beta readiness review.
- [x] Beta capability matrix exists at `docs/BETA_CAPABILITY_MATRIX.md`.
- [x] API inventory exists at `docs/API_INVENTORY.md`.
- [x] OpenAPI and AsyncAPI parse as YAML.
- [x] Examples validate through `./scripts/validate-examples.sh`.
- [x] Architecture boundaries validate through `./scripts/verify-architecture.sh`.
- [x] Secret scan validates through `./scripts/verify-no-secrets.sh`.
- [x] Full local verification passes through `make verify`.

## Capability Review

- [x] PipelineRun runtime is documented as a shell-based foundation.
- [x] DeploymentRun runtime is documented as guarded YAML/Kubernetes foundation.
- [x] GitOps/Argo CD sync is documented as disabled by default and guarded.
- [x] Artifact registry support is documented as OCI/Harbor-compatible foundation, not full registry management.
- [x] DevSecOps scanners/signers/SBOM are documented as foundation or future work.
- [x] Cloud and host deployment capabilities are documented as inventory/planning/guarded foundations.
- [x] Web UI is documented as a minimal foundation, not a complete product UI.

## API Review

- [x] Implemented, partial, and placeholder route groups are inventoried.
- [x] Structured `not_implemented` responses remain for placeholder route groups.
- [x] Pagination is opt-in and documented for runtime list/log/event/timeline reads and audit search.
- [x] Request/log/manifest size limits are documented.
- [x] Runner mutation endpoints use runner token validation or RBAC where applicable.

## Documentation Review

- [x] README links beta readiness artifacts.
- [x] Documentation index links beta capability and API inventory.
- [x] Roadmap documents Phase 9.0 beta freeze.
- [x] No production-ready or GA claims were added.
- [x] Known limitations are visible in beta release notes.

## Examples Review

- [x] Baseline examples are safe and credential-free.
- [x] Apply/sync/remote execution examples remain guarded by explicit flags or environment variables.
- [x] Optional local environments are documented as optional and not CI requirements.
- [x] Stale or future-oriented examples are marked by filename, placeholder values, or docs as example-only.

## Dependency Review

- [x] Direct runtime dependencies remain small: `chi`, `pgx`, `cobra`, and `yaml.v3`.
- [x] No heavy workflow engine was added.
- [x] No cloud, Kubernetes, Argo CD, registry, scanner, notification, or frontend-heavy backend SDK dependency is required for tests.
- [x] Web dependencies remain isolated under `web/`.

## Config Review

- [x] Local configs default to in-memory runtime and local object store for developer use.
- [x] `configs/production.example.yaml` uses placeholder internal hostnames and environment-variable token references only.
- [x] No kubeconfig, cloud credential, registry password, SSH key, or webhook secret appears in config examples.
- [x] Production-direction config still requires operator review before real deployment.

## Migration Review

- [x] Migrations are paired with up/down files through `000006_performance_indexes`.
- [x] Runtime persistence indexes cover common status, lease, log, event, audit, runner, and outbox paths.
- [x] Migration validation is included in example/quality checks.
- [x] Destructive schema changes are not introduced for beta freeze.

## Release Blockers

- [ ] Any failing `make verify` run.
- [ ] Any committed secret or realistic credential.
- [ ] Any production-ready or GA claim.
- [ ] Any baseline test requiring Kubernetes, cloud, registry, Argo CD, Vault, or external network services.
- [ ] Any public API behavior that is implemented but missing from OpenAPI.
- [ ] Any emitted event that is implemented but missing from AsyncAPI.

## Manual Validation Notes

- Docker build may require access to external base image registries.
- Helm checks require `helm` locally; `make verify` skips Helm lint/template only if Helm is unavailable.
- Load scripts require a running local server and are not CI-required distributed load tests.
- Real PostgreSQL, Kubernetes, Argo CD, Harbor, Vault, and cloud-provider checks remain optional manual validations.
