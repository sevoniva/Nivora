# Roadmap Overview

Nivora develops through explicit phases to avoid phase creep and premature implementation. Current work includes the Phase 1 shell-based PipelineRun runtime foundation and the Phase 2.1 Kubernetes YAML DeploymentRun runtime foundation.

## Phases

- [Phase 0: Skeleton](phase-0.md)
- [Phase 0.5: Guardrails and Validation](phase-0.5.md)
- [Phase 0.6: Planning and Collaboration Docs](phase-0.6.md)
- [Phase 1: Minimal Pipeline Execution](phase-1.md)
- Phase 1.5: Durable Runtime Foundation is tracked in the Phase 1 doc until it needs a separate page.
- Phase 1.6: Runtime Acceptance and Developer Experience is tracked in the Phase 1 doc until it needs a separate page.
- Phase 2.0 and Phase 2.1 are tracked in [Phase 2: Release and Deployment Foundation](phase-2.md).
- [Phase 3: Multi-Cloud and DevSecOps](phase-3.md)
- [Phase 4: Visualization Frontend](phase-4.md)

Phase docs describe objective, scope, non-goals, deliverables, acceptance criteria, and contribution opportunities.
