# v0.5.0-beta Release Notes Draft

Status: draft for beta freeze

Nivora `v0.5.0-beta` is intended to be a serious backend beta for contributors and platform engineers evaluating Nivora's delivery control-plane architecture. It is not a production GA release.

## Highlights

- Shell PipelineRun runtime with logs, events, audit, timeline, retry, timeout, cancellation, worker recovery, runner claim protocol, and performance benchmarks.
- Kubernetes YAML DeploymentRun foundation with render/validate/plan/dry-run, explicit guarded apply, inventory, health, diff, snapshots, and rollback planning.
- Release, ReleaseArtifact, ReleasePlan, and ReleaseExecution foundations for multi-target release orchestration.
- OCI artifact identity foundation with digest resolution, mutable-tag warnings, release binding, and Harbor-compatible OCI usage.
- GitOps and Argo CD foundation with plan/diff/status and guarded sync disabled by default.
- DevSecOps policy gate foundation with scan/finding models, fake/noop scanners, policy decisions, and audit/events.
- Secret/Credential, Auth/RBAC, Approval, Change Window, Cloud Inventory, Host Deployment, Compliance Audit, Plugin, Visualization, Observability, Packaging, and Performance foundations.

## Compatibility Notes

- API compatibility is beta-level and may still change before GA.
- Optional pagination uses `limit` and `offset`; legacy array responses are preserved when pagination is omitted.
- Configuration defaults remain developer-friendly and are not production defaults.
- Existing examples avoid committed credentials and external service requirements.

## Operational Notes

- `make verify` is the baseline local verification command.
- `make benchmark` runs local Go benchmarks for runtime and deployment planning paths.
- Load scripts in `scripts/load-*` require a running local server and are not distributed load tests.
- PostgreSQL migrations include runtime persistence, recovery, runner fleet, and performance index foundations.

## Known Limitations

- Nivora remains a hardened beta-candidate foundation and is not production-ready.
- Production Kubernetes semantics, destructive rollback, production Argo CD automation, cloud deployment, real Git provider integration, full registry management, and host SSH rollout hardening remain future work.
- External scanner/signer/SBOM, ITSM, notification, Vault/KMS, and enterprise SSO integrations are not complete.
- Multi-tenancy quotas and rate limits are foundational and not final production isolation.
- Web UI is a minimal foundation, not a complete product console.

## Verification Target

Before tagging beta, maintainers should run:

```bash
go mod tidy
gofmt -w on touched Go files
go test ./...
go vet ./...
go build ./cmd/nivora-server
go build ./cmd/nivora-worker
go build ./cmd/nivora-runner
go build ./cmd/nivora
./scripts/verify-architecture.sh
./scripts/verify-no-secrets.sh
./scripts/verify-api-specs.sh
make verify
git diff --check
```
